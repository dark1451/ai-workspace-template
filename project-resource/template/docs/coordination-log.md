# 조율 로그 (Coordination log)

> **역할 에이전트**(기획·디자인·개발·테스트)가 병렬/연속 세션에서 PM·다른 역할·사용자에게 남기는 **짧은 핸드오프**다.
> 채팅으로 세션끼리 말할 수 없으므로, **항상 이 파일 맨 아래에 새 블록을 추가**한다.
> 개념·운영 원칙: `docs/role-coordination.md`

---

## 기록 형식 (복사 후 맨 아래에 붙여 넣기)

```markdown
### YYYY-MM-DDThh:mm:ssZ | [역할: 한글역할명] | 세션 요약

- **delta**: `progress` | `none` | `blocked`  (진행 있음 / 보드·코드·문서 변화 없음 / 막힘)
- **tasks**: (예: `feat-001` in_progress → in_test 없음, 또는 "픽업 없음")
- **did**: 한 줄로 무엇을 했는지 (없으면 "없음 — 이유")
- **next**: 다음 세션(또는 다른 역할)이 하면 좋은 일 1~2줄
- **needs**: 사용자·PM·타 역할에게 필요한 것 (없으면 "—")
```
